Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una interfaz de programación que representa la estructura de un documento HTML o XML como un árbol, permitiendo acceder y manipular dinámicamente los elementos y atributos del documento. Es utilizado en el desarrollo web para crear interacciones dinámicas y modificar el contenido de una página en respuesta a eventos o acciones del usuario, brindando la posibilidad de realizar cambios en tiempo real en los elementos de una página web.

  2. ¿Qué es un nodo?

    3. En el contexto del DOM (Document Object Model), un nodo es un componente individual dentro de la estructura jerárquica de un documento HTML o XML. Los nodos representan elementos como etiquetas, texto, atributos o comentarios. Estos nodos se organizan en forma de árbol, donde cada nodo puede tener nodos hijos y hermanos. Cada nodo tiene propiedades y métodos para acceder y modificar su contenido, como obtener su etiqueta, atributos o texto.

  3. ¿Cómo se accede a un elemento del DOM?

    4. Para acceder a un elemento del DOM, se utilizan métodos getElementById, getElementsByClassName o querySelectorAll, getElementsByTagName o querySelectorAll, querySelector.

  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM, puedes utilizar las propiedades y métodos como innerHTML, attributes, getAttribute o setAttribute, style, classList. Puedes agregar, eliminar o reemplazar elementos y nodos en el DOM utilizando métodos como appendChild, removeChild o replaceChild.

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. Radica en la forma en que seleccionan los elementos del DOM, getElementBy* se centra en la selección de elementos por ID, clase o etiqueta, querySelector* permite una selección más avanzada utilizando selectores CSS, brindando mayor flexibilidad y precisión en la selección de elementos del DOM.

  6. ¿Qué es un evento?

    7. Un evento es una acción o suceso que ocurre en un elemento del DOM, como un clic del mouse, una pulsación de tecla o el cargado completo de una página. Los eventos permiten la interacción y la respuesta dinámica de una página web.

  7. ¿Qué es un listener?

    8. Una función o código que se encarga de detectar y responder a un evento específico que ocurre en un elemento del DOM. En resumen, un listener permite "escuchar" y responder a eventos específicos en un elemento del DOM mediante la ejecución de código predefinido.

  8. ¿Qué es un callback?

    9. Un callback es una función que se pasa como argumento a otra función y se ejecuta en respuesta a un evento o condición específica. El callback actúa como una devolución de llamada, permitiendo que la función receptora invoque el código del callback en el momento adecuado

  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento es un objeto proporcionado por el entorno de ejecución (como un navegador web) cuando ocurre un evento en el DOM. Contiene información sobre el evento en sí, como el tipo de evento, el elemento objetivo, las coordenadas del evento, entre otros detalles relevantes. El objeto de evento se pasa como argumento a los manejadores de eventos o callbacks, lo que permite acceder a dicha información y realizar acciones basadas en el evento que ocurrió. En resumen, un objeto de evento proporciona información contextual sobre un evento específico que ha ocurrido en el DOM.
